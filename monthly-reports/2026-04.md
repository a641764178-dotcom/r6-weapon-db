### 2.2 【建议修复】高置信度差异（按严重度排序）

> ⚠️ 以下仅是月度报告，不自动写入 data.js。等用户确认后再手动更新。

#### 🔴 P0 — 明显数据错误
| 武器 | 字段 | 当前值 | 建议值 | 依据 |
|---|---|---|---|---|
| UMP45 | damage | 38 | **42** | GitHub 实测 42 ∧ 灰机wiki 42；4/13 那次 42→38 的修改依据存疑，应回退 |
| UMP45 | falloff.min | 22 | **25** | floor(42×0.6)=25，当前是按 38 算出来的 22 |

#### 🟡 P1 — 小幅偏差（精度/口径差异）
| 武器 | 字段 | 当前值 | 建议值 | 依据 |
|---|---|---|---|---|
| M249 | falloff.min | 31 | 32 | GitHub 40m+ 实测 32（floor(48×0.65)=31，实际地板值被游戏上浮 1 点） |
| SR-25 | falloff.min | 42 | 43 | GitHub 40m+ 实测 43（floor(61×0.7)=42，同上属游戏地板值微调） |
| DP27 | falloff.min | 39 | 40 | GitHub 40m+ 实测 40（floor(60×0.65)=39） |

**规律**：MEMORY.md 里的公式 `min = floor(damage × 系数)` 在 AR/SMG 类上吻合良好，但 LMG/DMR/狙击类经常被游戏内上浮 1 点（实际最低值 = `floor(damage × 系数) + 1`）。**建议在 MEMORY.md 里补一条"LMG/DMR/狙击类 min 可能比公式 +1"的注释**，并在这三类武器的 falloff.min 字段上统一做修正。

#### 🔵 P2 — 口径差异（不算错误，可保留）
| 武器 | 字段 | 当前值 | 实测值 | 说明 |
|---|---|---|---|---|
| SR-25 | rpm | 439 | 445 | 灰机 ≤439（理论点击上限）；GitHub 445（实测秒发均值）。**保持 439**，与灰机一致 |
| TCSG12 | rpm | 487 | 496 | 同上，**保持 487** |
| M870 | rpm | 0 | 102 | 语义差异：0 表示"非自动"；若要补数值建议 100（与灰机 ≤100 一致） |
| SG-CQB / 多数霰弹 | rpm | 0 | 88/102 | 同上 |

```

AK-12   : FAIL HTTP 567
```

### 4.2 浏览器视角验证
使用浏览器渲染工具（带标准 UA + Referer: `r6s.huijiwiki.com`）访问 M4 后坐力图：**成功返回图片元数据 341×376 jpg**。

**结论**：
- CDN **对无 Referer 的服务器端直连做了 WAF 拦截**（返回 567）
- **对带 Referer 的浏览器端正常放行**
- 现网用户访问 `https://a641764178-dotcom.github.io/r6-weapon-db/` 时，浏览器会自动带上 `Referer: github.io`，而根据 MEMORY.md 的经验，灰机 CDN 对 github.io 这种跨域 Referer 是否放行存在风险

### 4.3 🟡 P1 建议：后坐力图可用性监控
1. 所有 `<img>` 标签已按 MEMORY.md 要求加 `referrerpolicy="no-referrer"`——但这实际上会**去掉** Referer，理论上可能触发 CDN 保护
2. 建议在下次人工访问线上页面时，打开 F12 Network 确认 Backlash_*.jpg 加载状态码
3. 如发现批量 567/403，应考虑**迁移到本地仓库**（把 95 张 Backlash 图 pull 下来放到 `r6-weapon-db/images/recoil/`）
